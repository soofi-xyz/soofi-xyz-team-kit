# Initiative notes

## Headings

Use plain labels without markdown `##`:

- `Context` — business pressure, transcript/PDF source, urgency
- `Description` — scope of work in one short paragraph
- `Acceptance Criteria` — indented plain bullets (not markdown checkboxes in project notes)

## Acceptance criteria quality

Each AC should be verifiable within the initiative timeframe.

**Good:** `Configure 15 percentage-point reduction in offer engine for eligible accounts.`

**Bad:** `Improve settlements.` (not verifiable)

## Tiering war-plan work

When drafting from planning docs:

- **Near-term focus** — bold PDF items or explicit transcript urgency (launch today, 2-day)
- **Backlog** — everything else; still draft full Context/Description/AC

Flag overlaps with existing portfolio projects: create new vs update existing vs skip.

## AC tasks in Asana

After project notes are set, seed **Acceptance Criteria** custom-type tasks by duplicating an existing AC task from the AVE template project, then edit titles per criterion.
