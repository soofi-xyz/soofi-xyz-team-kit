# Section Definitions

Rules are organized by filename prefix. Each prefix maps to a section:

| Prefix | Section | Priority |
| --- | --- | --- |
| `catalog-` | Data monitoring and change detection | 1 — CRITICAL |
| `fact-` | Fact generation and evidence | 2 — CRITICAL |
| `approval-` | Human approval in Asana | 3 — CRITICAL |
| `publish-` | Content-only website publishing | 4 — CRITICAL |
| `operations-` | Reliability, observability, verification | 5 — HIGH |
